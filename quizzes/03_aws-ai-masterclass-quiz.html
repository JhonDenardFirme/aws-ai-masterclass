MC 1MCA2
A product team argues Responsible AI only matters at deployment — "we ship it responsibly, the rest is fine." What is the correct response?
ACorrect — deployment is when the model interacts with real users, making it the only phase with measurable impact.
BResponsible AI applies at all five stages: Design, Development, Deployment, Monitoring, and Evaluation. Bias introduced at data collection cannot be fixed at deployment alone.
CResponsible AI applies only at Design and Deployment — the two stages where ethical decisions and outputs occur.
DResponsible AI applies only post-deployment during Monitoring, when real-world harms first become detectable.
MC 2MCB1
Model A: 97% training accuracy, 58% test accuracy. Model B: 74% training, 72% test. Which diagnosis and recommendation is correct?
AModel A is better — higher training accuracy signals stronger pattern learning. Discard Model B.
BModel A is overfitting (low bias, high variance) — memorized noise. Model B is balanced (consistent train/test) — better generalization. Prefer Model B for production.
CBoth models are underfitting — neither achieves acceptable training accuracy.
DModel A is underfitting (high bias) — the train/test gap is the signature of an overly simple model.
MC 3MCB2
A deep learning model overfits on a medical dataset. No additional data can be collected. Which two mitigations are most applicable without requiring more data?
AIncrease Data and Cross-Validation — more samples and fold rotation directly address memorization.
BRegularization and Early Stopping — Regularization penalizes complexity; Early Stopping halts before memorization deepens. Neither requires additional data.
CDimensionality Reduction and Increase Data — reduce features and collect more samples.
DCross-Validation and Increase Data — rotating splits and collecting more scans will solve overfitting.
MC 4MCB3
A legal AI confidently cites a 2021 Supreme Court ruling — case name, judge, ruling language all internally consistent — that does not exist. Which Generative AI challenge is this?
AToxicity — the fabricated ruling could damage the user's legal standing.
BIntellectual Property — the model reproduced copyrighted content from a real ruling in its training data.
CHallucination — plausible but entirely fabricated information presented with confidence.
DNondeterminism — the model produces different citations each time the same question is asked.
MC 5MCC1–C2
A bank's credit model has documentation: trained on 10 years of data using gradient boosting with listed hyperparameters. But when a customer asks why their loan was denied, no reason can be provided. Which dimension is satisfied and which is missing?
AFairness satisfied (documented process); Safety missing (no human oversight on denials).
BTransparency satisfied (HOW the model works is documented); Explainability missing (WHY this specific decision was made cannot be provided).
CExplainability satisfied (algorithm documented); Transparency missing (process not visible to regulators).
DGovernance satisfied (documentation exists); Explainability missing (customers cannot understand outcomes).
MC 6MCC1

ATransparency — the models' internal processes are not visible to affected individuals.
BFairness — both systems produce biased, discriminatory outcomes across demographic groups.
CGovernance — the organizations failed to enforce policies preventing biased deployment.
DControllability — neither system can be steered to produce unbiased outputs in production.

CAmazon Bedrock — provides the evaluation framework for all ML models via its automatic evaluation pipeline.
DBoth equally — SageMaker trains, Bedrock deploys and monitors.
MC 8MCD3

ASageMaker Model Monitor — tracks outputs and flags policy violations in production.

CSageMaker Clarify — detects and removes policy-violating content using feature-level analysis on chatbot outputs.
DFine-tuning the FM — training on compliant examples eliminates the need for runtime content controls.
MC 9MCD4
A credit dataset has 95,000 approved records and 5,000 denied records. A data scientist wants to balance it without simply duplicating the 5,000 denied records. Which Data Wrangler technique applies?
ARandom Oversampling — duplicate the 5,000 denied records until classes are balanced.
BRandom Undersampling — remove approved records until the dataset is balanced.
CSMOTE — create synthetic denied records by interpolating between existing minority samples, producing diverse new examples without duplication.
DSageMaker Clarify — detect the imbalance and auto-generate balanced training splits.
MC 10MCD5

AAmazon A2I — routes flagged predictions to human reviewers when data patterns shift.
BSageMaker Clarify — re-runs bias detection on incoming requests to flag demographic shifts.
CSageMaker Model Monitor — tracks production performance, detects data drift when input distribution shifts from training data, and sends anomaly alerts.
DSageMaker Model Dashboard — provides centralized risk ratings and surfaces data quality issues.
MC 11MCE1

APerformance — GPU-hour requirements are a latency and throughput concern under performance factors.
BSustainability — energy consumption, carbon impact, and hardware demand are sustainability factors in responsible model selection.
CConstraints — infrastructure demand falls under deployment constraints.
DEconomic Considerations — GPU costs are a financial concern under the economics dimension.
MC 12MCF1
A facial recognition system trained exclusively on adult images consistently misidentifies children. The team proposes improving the algorithm. What is the root cause and correct intervention?
AInsufficient model complexity — use a deeper network to capture child facial features.
BDataset bias — children were absent from training data so the model never learned their features. Intervention: inclusive data collection including children, then retrain.
COverfitting to adult features — apply regularization to prevent over-specialization on adult patterns.
DHigh variance — apply early stopping to prevent memorization of adult-specific noise.
MC 13MCG3
A denied mortgage applicant is told: "Your income contributed 0.58 to the denial score, employment duration 0.31, and debt-to-income ratio 0.24." Which explainability technique produced this?
ACounterfactual — identifies what the applicant would need to change for approval.
BLIME — a local model was built around this specific application to approximate the black box's decision.
CSHAP — feature contribution scores assigned to each input for this specific prediction using game theory.
DDimensionality Reduction — the model reduced the application to its three most influential features.
MC 14MCG3
A denied applicant is told: "If your annual income were $12,000 higher and outstanding debt reduced by $8,000, your application would have been approved." Which explainability technique is this?
ASHAP — contribution scores show income and debt as the top-weighted factors.
BLIME — a local approximation identified income and debt as the most influential features in this region.
CCounterfactual — answers "what would need to change for a different outcome?" with specific, actionable changes.
DCross-Validation — fold analysis identified which input configurations lead to approval vs. denial.
MC 15MCG4

AFalse Transparency — SHAP scores approximated but didn't accurately represent the model's actual process.
BHigher Cost — SHAP overhead degraded performance, creating exploitable latency windows.
CSecurity Risk — feature importance scores revealed model internals enabling adversaries to craft targeted evasion attacks.
DPrivacy Exposure — SHAP scores inadvertently revealed personal information about training data individuals.
MC 16MCH1

AModel B — 12% accuracy advantage outweighs the explainability requirement; judges can trust the recommendation.
BModel A — the legal explainability mandate is a hard constraint. Compliance cannot be traded for performance. This is the interpretability-performance trade-off resolved by regulation.
CModel B with LIME post-hoc explanations — approximation techniques satisfy the legal requirement while preserving accuracy.
DNeither — train a new model achieving both 91% accuracy and full explainability before deployment.
MC 17MCH1
A content moderation system filters so aggressively it blocks 40% of legitimate customer service responses. Which model trade-off does this represent?
AInterpretability vs. Performance — the explainable filter logic is too simple to distinguish harmful from helpful content accurately.
BFiltering vs. Usefulness — aggressive safety filtering reduced harm but over-blocked legitimate outputs, degrading model utility. The filter threshold needs recalibration.
CPrivacy vs. Transparency — filter audit logs expose customer interaction data.
DSecurity vs. Auditability — moderation logs create security risk while being required for compliance audits.

TOF 1T/FA2
Responsible AI applies primarily during the Deployment stage, since that is when the model interacts with real users and causes real-world impact.
TOF 2T/FB1
A model with high bias and low variance is underfitting — it is too simple to capture the true pattern in the data.
TOF 3T/FB1
Overfitting occurs when a model achieves excellent performance on both training and test data simultaneously.
TOF 4T/FB2
Early Stopping prevents overfitting by halting training when validation performance begins to deteriorate, before the model memorizes training-specific noise.
TOF 5T/FB2
Regularization addresses overfitting by adding more training data to expose the model to greater variety.
TOF 6T/FB3
Hallucination in Generative AI means the model produces outputs that are plausible-sounding but factually false — because it predicts likely text, not correct text.
TOF 7T/FC1–C2
Transparency and Explainability are interchangeable terms in Responsible AI — both refer to understanding why a model made a specific decision.
TOF 8T/FC1
No single Responsible AI dimension is sufficient on its own — all eight must be addressed for fully responsible AI.
TOF 9T/FD1
Amazon Bedrock is the correct AWS platform for detecting bias in a custom-built ML model using feature-level analysis.
TOF 10T/FD3
Guardrails for Amazon Bedrock apply across all Foundation Models accessed through Bedrock — they are not specific to any individual model.
TOF 11T/FD4
SMOTE balances datasets by duplicating existing minority class samples until the classes are equal.
TOF 12T/FD5
SageMaker Model Monitor detects data drift and sends anomaly alerts when a production model's input distribution shifts from its training distribution.
TOF 13T/FE1
Model performance is absolute — a model achieving 95% accuracy on a benchmark will achieve approximately the same on any real-world dataset in the same domain.
TOF 14T/FF1
Data bias directly causes model bias — no algorithm, regardless of sophistication, can fully compensate for a fundamentally biased training dataset.
TOF 15T/FG3
LIME produces globally accurate explanations by analyzing feature contributions across the entire model using game theory to assign credit to each input variable.
TOF 16T/FG4
Publishing detailed SHAP feature importance scores publicly can create a security risk by revealing model internals that adversaries can use to craft evasion attacks.
TOF 17T/FH1
Choosing a more interpretable model generally requires some sacrifice in accuracy on complex tasks — the interpretability-performance trade-off is a real design tension.
TOF 18T/FI3
RLHF aligns AI behavior with human values by using human preference ratings to train a reward model, which then guides FM fine-tuning.
TOF 19T/FI1
The Reflexivity principle of Human-Centered AI Design means the system should automatically revise its own outputs before presenting them to users.
TOF 20T/FI3
SageMaker Ground Truth is the AWS tool used to collect and manage human feedback for RLHF workflows.
